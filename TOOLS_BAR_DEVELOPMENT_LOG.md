## 1. 🗂️ 项目架构与核心理念

- **架构模式**：纯前端 (Vanilla JS + HTML/CSS) + 局部 Vue3 (复杂组件)。
- **核心理念**：Local First（本地优先），零后端依赖，所有数据缓存在 `localStorage`。
- **UI 机制**：Dashboard 小组件 (Widget) 预览实时数据 + 沉浸式 iframe 弹窗 (`dialog`) 承载重度应用。

### Phase 3: 工具库扩充与路径纠正

### Phase 4: 整合 EV 续航与成本计算器
- **动作**：从外部文件夹提取最新的 `energy&cost cal.html`，重命名为 `ev_energy_cal.html`。
- **适配**：在 `app.js` 中新增 `evEnergyCal` 的小组件 UI（包含 4 个数据块和渐变色设计），更新默认固定列表。

### Phase 5: UX 体验升级 (弹窗交互)
- **修复**：全屏弹窗会在外部面板刷新时意外关闭。
- **优化**：
  1. 封装统一的 `closeTool()` 逻辑。
  2. 增加点击灰色遮罩层 (`e.target === dialog`) 关闭弹窗功能。
  3. 增加键盘 `ESC` 键关闭弹窗功能。

### Phase 6: 家庭资产模型 v2.0 重大升级
- **动作**：引入 `family weathy/update/` 下的最新的 v9 拆分版代码（包含配置导出导入、单双收入切换、500-5000次动态仿真）。
- **重构**：将老旧的 `asset_sim.html` 替换为 `family_wealth_sim_v2.html`，同步拷贝关联的 `js` 与 `css`，并更新 Dashboard 小组件描述和配置。

### Phase 7: GitHub Pages 部署与安全策略适配 (HTTPS/CORS)
- **痛点**：部署到 GitHub Pages 后，出现 `Mixed Content`、`ERR_CERT_COMMON_NAME_INVALID` 以及 `sandbox` 逃逸警告。
- **核心修复**：
  1. **HTTP -> HTTPS**：将腾讯股票接口 `http://qt.gtimg.cn` 强制升级为 `https://`，解决混合内容拦截。
  2. **防 DNS 污染**：将币安 API `api.binance.com` 替换为官方无限制备用域名 `data-api.binance.vision`，彻底解决国内直连的证书无效问题。
  3. **解除沙盒束缚**：移除 iframe 的 `sandbox` 属性，因为所有工具均为本地受信任代码，同源且需执行 JS，以此消除浏览器的安全警告。

## 3. 🌐 高价值 API 资产库 (前端直连免跨域)

在迭代中探索出的**支持纯前端直连（带 `Access-Control-Allow-Origin: *`）、免费且无需后端的 API**：

1. **全球股票指数 (腾讯)**
   - URL: `https://qt.gtimg.cn/q=s_sh000001,s_sz399001,s_us.NDX,s_hkHSI`
   - 特点：支持批量查询，解析正则 `v_s_代码=".*?~名称~代码~价格~涨跌额~涨跌幅~`。
2. **加密货币 (Binance Vision - 避开国内墙)**
   - URL: `https://data-api.binance.vision/api/v3/ticker/24hr?symbols=["BTCUSDT","ETHUSDT"]`
   - 特点：极度稳定，不报证书错误，数据绝对实时。
3. **全球汇率 (ExchangeRate-API)**
   - URL: `https://api.exchangerate-api.com/v4/latest/USD`
   - 特点：每日更新，基础汇率换算免费。

```markdown

- 如果报 `CORS` 或 `Failed to fetch`：请寻找支持 `Access-Control-Allow-Origin: *` 的免费接口替代方案。
- 如果报 `ERR_CERT_COMMON_NAME_INVALID` (尤其是加密货币)：请考虑 DNS 污染问题，尝试更换备用域名 (如 api.binance.com 换为 data-api.binance.vision)。
```
